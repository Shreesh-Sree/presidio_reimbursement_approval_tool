# Supabase OAuth setup

This legacy document no longer names a production project, domain, or OAuth
client. Those values are environment-specific and must never be copied blindly
between staging and production.

Use [Supabase/Auth production promotion](docs/supabase-production-promotion.md)
instead. It requires exact redirect URLs (not wildcard patterns), approved
provider callbacks, hosted-console verification, and redacted deployment
evidence.
